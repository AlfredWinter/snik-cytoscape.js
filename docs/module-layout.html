layout

Due to JavaScript being a slow mostly single-threaded language with no really fast layouting library available, layouting the full 4000+ node graph can take a minute or more depending on the client PC. After the first time, the layout is cached and reused, until major changes occur in the graph. If a breakthrough occurs in JavaScript graph layouting, update here and possibly remove cache.

(static) presetLayout(cy, pos)

Source:
  • layout.js, line 81
Applies a preset layout matching the node id's to the first element of each subarray in pos. Nodes without matching entry in pos are set to position {x:0,y:0}, positions without matching node id are ignored.
Example
presetLayout(cy,[["http://www.snik.eu...",{"x":0,"y":0}],...]);
Parameters:
Name Type Description
cy cytoscape the Cytoscape.js graph to apply the positions on, node id's need to match those in the given positions
pos array an array of arrays, each of which contains the positions for a node id
Returns:
whether the layout could be successfully applied

(static) run(cy, layoutConfig, subs)

Source:
  • layout.js, line 47
Layouts all visible nodes in a graph. Saves to cache but doesn't load from it (use module:layout.runCached for that).
Example
run(cy,{"name":"grid"},new Set(["meta","ciox"]))
Parameters:
Name Type Description
cy cy.cytoscape the Cytoscape.js graph to run the layout on
layoutConfig json the layout configuration, which includes the layout name and options
subs Set Set of subontologies. If the subs are not given the layout still works but it is not cached.
Returns:
whether the layout could successfully be applied. Does not indicate success of saving to cache.
